This group project involved building a machine learning analysis pipeline to predict diseases from patient-reported symptoms across 41 disease classes and 132 binary symptom features. The dataset was open source and obtained from Kaggle.
My contributions spanned both technical and coordination work. 
On the technical side, I implemented and evaluated the following models: Logistic regression, XGBoost, and K-Nearest Neighbors. I explored the hyperparameter space to optimize models through grid search tuning for XGBoost and systematic K-curve and distance metric analysis for KNN. I also conducted SHAP-based interpretability analysis to identify the most diagnostically impactful symptoms and developed a comparison framework to run evaluations across all candidate models and compile comparative results and visualizations.
Beyond contributing to the analysis, I took on an active coordination role across the team to track progress, align on project activities & timelines, and support teammates with debugging any issues.
I also maintained the project README file, integrating each team member's results, visualizations, and model write-ups into a single cohesive final report.
Given additional time, I would pursue three directions. Most importantly, testing all models on real clinical data to assess the performance of the models under realistic, noisy conditions. I would also extend the SHAP analysis to a per-disease level to identify the most discriminative symptom for each of the 41 conditions individually. Finally, I would build a simple interactive symptom checker as a proof-of-concept clinical tool, allowing a user to input symptoms and receive a ranked list of probable diagnoses with confidence scores.
